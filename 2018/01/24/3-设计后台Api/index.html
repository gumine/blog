设计后台Api

对于签到系统来说，后台的Api主要包括两方面，一对已有数据的查询，二添加新的数据。

据此我们设计以下三个Api：

  • /api/Record GET 查询数据 参数:begin,end
  • /api/Record/SignIn POST 新建一条签到记录 参数:user,seat
  • /api/Record/SignOut POST 记录签退时间 参数:user
