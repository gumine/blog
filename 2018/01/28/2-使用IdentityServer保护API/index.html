使用IdentityServer保护API

在经过上一节之后，我们就已经有了一个可用的IdentityServer，这节我们将创建一个由IdentityServer保护的API服务。

API构建

因为是使用VS开发，我们直接在解决方案中创建一个新的WebAPI项目。

添加IdentityServer4.AccessTokenValidation引用。

然后，在Startup.cs中配置认证服务。

之后，我们只需要给我们想要保护的Controller（或者Action）加上[Authorize]属性即可。

最后，在启动前，我们还是要设置一下绑定的端口，这次我们将API绑定到5001端口。

启动后，我们可以通过Postman来试着访问一下API，发现返回的是401(Unauthorized)，说明，我们的API接口已经是需要验证后才能访问。
